# Video 前端模块：设计目标与职责边界

## 1. 设计目标

### 1.1 提供独立的视频总结操作界面

在 Studio 面板中新增 Video 视图板块，用户无需离开 notebook 工作区即可完成视频总结的触发、进度追踪和结果浏览。Video 视图与 Notes、Diagrams 在交互层级和导航模式上保持一致，降低用户的认知负担。

### 1.2 输入优先的交互范式

与 Notes 的「列表优先」不同，Video 视图将 URL 输入区域置于顶部最显著位置。这反映了 Video 模块的核心动作是「输入 URL -> 获得总结」，而不是「浏览已有内容」。历史摘要列表位于输入区域下方，作为辅助而非主体。

### 1.3 实时反馈的总结进度

视频总结是一个耗时操作（通常 10-60 秒）。前端必须在总结过程中持续向用户反馈当前阶段，而不是让用户面对一个无响应的等待状态。进度展示采用 step indicator 方式，在输入区域内联显示当前步骤（获取视频信息、获取字幕、AI 生成总结等），出错时就地展示错误信息和重试入口。

### 1.4 双通道的结果汇聚

用户可以通过两条路径触发视频总结：Video 面板直接操作（独立 SSE）和主聊天面板 `/video` 命令（agent loop）。无论哪条路径产生的结果，都应该在 Video 面板的历史列表中可见。前端通过 TanStack Query 的缓存失效机制实现这一点，不需要两条路径之间的显式协调。

### 1.5 与现有 Studio 体系的一致性

Video 前端模块在以下方面与现有 Notes、Diagrams 保持一致：

- 视图导航模式（list view + detail view）
- 状态管理位置（studio-store.ts）
- 数据获取模式（TanStack Query hooks + API client）
- 删除确认交互（ConfirmDialog 组件）
- 国际化方式（uiStrings 中的双语键值对）
- 斜杠命令提示（SlashCommandHint 组件）

## 2. 职责

### 2.1 视频总结的触发与进度展示

接收用户输入的 Bilibili URL 或 BV 号，通过 API client 发起总结请求，建立 SSE 连接接收进度事件，在 UI 中实时展示当前步骤和状态。总结完成后刷新列表数据。

### 2.2 视频摘要的列表浏览与详情查看

展示用户的视频摘要历史（默认显示全部摘要，支持按当前 notebook 筛选）。每个列表条目展示缩略图、标题、UP 主和视频时长。提供从列表到详情的导航，详情视图以单列滚动方式展示完整信息。

### 2.3 视频摘要的 Notebook 关联管理

在详情视图中提供关联/取消关联到当前 notebook 的操作入口。关联状态的变更通过 API 调用完成，成功后刷新相关缓存。

### 2.4 视频摘要的删除

在详情视图中提供删除入口，通过 ConfirmDialog 确认后执行删除操作，完成后返回列表视图并刷新数据。

### 2.5 B站登录状态管理

在 Video 面板内展示 B 站登录状态（未登录/已登录）。未登录时提供登录入口，通过弹窗展示 QR 码。未登录时不阻断总结提交，当后端返回需要登录的错误时引导用户登录。

### 2.6 Studio 导航扩展

扩展 StudioView 类型和 studio-store.ts，新增 `"videos"` 和 `"video-detail"` 两个视图状态及相应的导航方法。在 Home 视图中新增 Video 卡片（第二行，保持现有 2 列网格）。

### 2.7 斜杠命令扩展

在 SlashCommandHint 组件的命令列表中新增 `/video` 条目及其描述文案，使用户在聊天输入框中输入 `/` 时能看到 video 命令选项。

### 2.8 国际化文案

在 uiStrings 中新增 `video` 命名空间，覆盖 Video 模块所有 UI 文案的中英文翻译。同时在 `slashCommand` 和 `confirmation` 命名空间中补充 Video 相关条目。

## 3. 非职责

- 不负责视频总结的业务逻辑（字幕提取、ASR 转录、LLM 调用均在后端完成）
- 不负责 SSE 事件的协议定义（遵循后端定义的事件格式）
- 不负责 Bilibili API 的调用（所有平台交互通过后端代理）
- 不负责 agent loop 的执行控制和工具调度（属于 chat-store 和后端 core/engine）
- 不负责 TanStack Query 的全局配置（复用现有 QueryClient 设置）
- 不负责 Markdown 渲染引擎的实现（复用现有渲染组件）
- 不实现视频播放器或视频预览功能（产品定位是笔记工具，不是视频播放器）
- 不实现批量总结、批量删除等批量操作（第一阶段仅支持单视频操作）
- 不实现视频摘要的本地编辑（总结内容由 LLM 生成，用户只读）
