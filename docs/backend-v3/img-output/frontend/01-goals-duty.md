# 图片生成模块（前端） - Goals & Duties

## Design Goals

1. 生成的图片在聊天界面上以结构化卡片形式呈现，紧随 assistant 消息文字之后，不嵌入 Markdown 内容流。

2. 图片卡片贴合图片实际尺寸渲染，提供点击放大和下载功能，交互轻量但不缺失。

3. SSE 事件到达后立即渲染卡片骨架，图片数据通过后端 API 按需加载，与现有流式消息渲染管线无缝衔接。

4. 不改变现有前端框架结构和组件模式，仅在已有 MessageItem 渲染流程中追加图片展示区域。

## Duties

- 在 SSE 事件处理中识别 `image_generated` 事件，将图片元数据追加到当前 assistant 消息的 `images` 字段。

- 渲染 `ImageCardList` 组件，对每张图片生成 `ImageCard`：图片 src 指向后端 `/api/generated-images/{imageId}/data`，右上角下载按钮，底部单行 prompt 省略显示。

- 提供 `ImageLightbox` 全屏查看组件，点击图片打开，ESC 或点击背景关闭。

- 加载状态处理：图片加载中显示骨架屏占位，加载失败显示失败占位和重试。

- 历史消息加载时，从后端消息 API 返回的 `images` 字段中读取图片元数据，渲染已完成生成的图片。

## Non-Duties

- 不负责图片生成请求的发起：图片生成由后端 Agent 工具调用触发，前端仅消费结果。

- 不负责图片存储或删除：图片的持久化和生命周期管理由后端负责，前端仅通过 API 获取。

- 不负责 prompt 优化：图片卡片显示的 prompt 是 Agent 优化后的绘图描述，前端不做改写。

- 不实现图片编辑功能：当前版本仅支持查看和下载，图片编辑不在前端职责范围内。

- 不实现图片画廊或批量管理界面：图片在聊天流中按消息组织，不需要独立的图片管理页面。
