# Markdown 查看器 -- 目标与职责

## 1. 设计目标

### 1.1 核心目标

将后端返回的 Markdown 文本渲染为可读性良好的 HTML 内容，支持 MinerU 转换产出的各类 Markdown 元素，并为用户提供接近 VS Code / Cursor 内置 Markdown Preview 的阅读体验。

### 1.2 具体目标

- 准确渲染：正确处理 GFM（表格、删除线、任务列表）、LaTeX 数学公式、代码块语法高亮、图片、引用块等常见 Markdown 元素
- 图片可达：后端在文档处理阶段已将 Markdown 中的图片路径转换为 API 绝对路径（`/api/v1/documents/{id}/assets/images/{hash}.jpg`），前端无需额外处理路径，浏览器可直接加载
- CJK 友好：中文内容具备合理的排版效果，包括正确的断行、标点处理和字体渲染
- 主题适配：内容区排版样式跟随应用的 light/dark 主题切换
- 性能可接受：对于 MinerU 转换产出的长文档（2000-3000 行 Markdown），渲染后页面可正常滚动浏览，不出现明显卡顿

---

## 2. 职责

### 2.1 Markdown 解析与渲染

接收 Markdown 字符串作为输入，通过 unified/remark/rehype 处理管线将其转换为 React 组件树并渲染到 DOM。管线包含以下处理能力：

- GFM 扩展语法解析（表格、删除线、自动链接、任务列表）
- LaTeX 数学公式渲染（行内公式和块级公式）
- 代码块语法高亮（基于 highlight.js）
- 标题锚点生成（用于目录跳转）

### 2.2 内容区排版样式

提供一套完整的 Markdown 内容排版 CSS 样式，覆盖所有标准 Markdown 元素的视觉表现：

- 标题层级（H1-H6）的字号、间距、字重
- 段落行高与段间距
- 代码块的背景、边框、语言标签
- 表格的边框、表头样式、条纹背景
- 引用块的左侧边线与背景
- 列表的缩进与标记样式
- 图片的居中与最大宽度限制
- 水平分割线样式

排版风格参照 VS Code / Cursor 内置 Markdown Preview，不追求学术期刊风格。

### 2.3 主题响应

内容区样式通过 CSS 变量与应用主题系统对接，在 light/dark 切换时自动更新：

- 背景色、文字色、链接色
- 代码块背景色与语法高亮配色
- 表格边框色与表头背景色
- 引用块边线色

---

## 3. 非职责

以下功能明确不属于本模块的职责范围：

- **Markdown 编辑**：本模块仅负责渲染，不提供编辑能力。如需笔记编辑功能，应由独立模块承担
- **文档内容获取**：从后端 API 获取 Markdown 文本的逻辑属于状态管理与 API 层，本模块只接收已获取的字符串
- **文本选择交互**：用户在渲染区域内选中文本后的浮动菜单和操作逻辑属于文本选择交互模块
- **目录导航（TOC）**：目录面板的展示和跳转交互不在本模块范围内。本模块仅生成标题锚点 ID 供外部使用
- **虚拟滚动与增量渲染**：MVP 阶段不实现，使用 `content-visibility: auto` CSS 属性做基础懒渲染。后续如有性能瓶颈再作为独立优化方案处理
- **多主题预设**：只实现 light/dark 两套配色，不实现类似 markdown-viewer-extension 的 29 种主题预设
- **图表与流程图渲染**：MinerU 转换产出的 Markdown 中暂未发现 Mermaid 或 Vega 等图表语法，不在 MVP 范围内。管线架构应允许后续添加此类插件
- **DOCX 导出**：不属于前端渲染模块的职责
